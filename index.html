Quick facts

LLM-GGP lives in llmggp/ and ships:

  • LLM scoring with caching (llmggp/api.py) plus a composable base prompt.
  • LLM-biased crossover and guarded mutation (llmggp/operators.py).
  • Shared base primitives (llmggp/pset_base.py) for your task-specific GP sets.
  • Fitness shaping (llmggp/fitness.py) with DEFAULT_ALPHA=0.2.

What is LLM-GGP?

LLM-GGP is a reusable large-language-model-guided genetic programming toolbox. Subtree semantics scored by an LLM influence crossover, mutation, and fitness shaping to evolve interpretable heuristics for any GP task.

Compose your own domain prompt, plug in a task-specific primitive set, and let LLM-scored subtrees steer variation and selection with DEFAULT_K_SELECT=3 and DEFAULT_ALPHA=0.2.

Core modules

Source lives in llmggp/; add your own task modules alongside.

LLM scoring pipeline

build_llm_client and llm_score_branch call a base prompt (BASE_SYSTEM_PROMPT) that can be extended via compose_system_prompt, returning a 0-1 score per subtree with caching and graceful degradation.

  • Env vars: LLM_HOST / LLM_API_KEY / LLM_MODEL.
  • No credentials? return 0.0 and keep the run offline-safe.
  • LRU cache up to 5000 entries to avoid duplicate calls.

LLM-biased operators

mate_llm_biased and mut_llm_guarded keep the highest-scoring deep subtrees and mutate weaker ones.

  • Select the k_select=3 deepest slices by default.
  • Higher score lowers mutation chance; crossover prefers high-score slices.
  • Fallback: mate_nonllm_subtree for pure GP baselines.

Primitive sets

add_basic_primitives attaches arithmetic/logic primitives; extend with your domain-specific terminals and primitives.

  • Works with gp.PrimitiveSetTyped out of the box.
  • Keep task-specific nodes in your own module (e.g., your_task/pset.py).

Fitness shaping

eval_with_llm_shaping combines simulation cost with LLM importance (shaped = cost * (1 - alpha * mean(score))).

  • DEFAULT_ALPHA=0.2 controls how much LLM guidance matters.
  • DEFAULT_K_SELECT=3 focuses on the deepest subtrees.
  • Stores individual.sim_score for later reporting.

Workflow

1Define your primitive set: start with add_basic_primitives(pset), then add domain nodes.
2Compose a domain prompt with compose_system_prompt.
3Wire scorer: partial(llm_score_branch, system_prompt=prompt, client=build_llm_client()).
4Run your DEAP loop with LLM-biased operators and shaped fitness.

Set LLM_HOST/LLM_API_KEY/LLM_MODEL for live calls; offline returns 0.0 scores.

LLM prompt (excerpt)

Role: Score GP subtrees in [0.0, 1.0] for your domain.
Primitives: define in your pset (e.g., add/sub/mul/div/ifelse + domain features).
Criteria: relevance, discriminativeness, simplicity, robustness, balance.

Example subtree

# String sent to llm_score_branch
tree = "add( RI(bay, seq), EMP(bay, seq) )"

# Fitness shaping (alpha = 0.2)
shaped = sim_cost * (1 - alpha * mean(scores))

Quick start

Steps

  • Install deps: pip install deap numpy (add tqdm/notebook if needed).
  • Optional: set LLM_HOST/LLM_API_KEY/LLM_MODEL to enable real scoring.
  • Build your pset + prompt, register scorer/ops in a DEAP toolbox.
  • Run your GP loop; log population, hall-of-fame, and scores as needed.

Without credentials it falls back to pure GP for offline debugging.

Key files

  • llmggp/api.py: LLM client and scoring cache.
  • llmggp/operators.py: LLM-biased crossover/mutation.
  • llmggp/fitness.py: Fitness shaping.
  • llmggp/psets.py: CRP primitive definitions.
  • util/util.py: Data IO and seed helpers.
